Project Objectives

The project integrates all required tasks into a single robust application, prioritizing efficiency and reliable safety actuation, demonstrating advanced embedded programming concepts like state machines, PWM, and I/O management.

Automation

Interaction

Overcoming Challenges

Engineering and technical problems encountered during development and how they were resolved.

A

Sensor Calibration

Challenge: Analog sensors (LDR, MQ-2) provide raw 0-1023 data that doesn't correspond directly to standardized real-world units or binary safe/unsafe states.

Solution: Implemented mapping algorithms using the map() function to convert raw luminosity data for PWM control and established practical empirical thresholds for gas and distance metrics through repetitive testing.

B

Button Debouncing

Challenge: Mechanical buttons introduce transient noise or "bouncing" when pressed, sending rapid multiple signals to the Arduino, leading to unintended and repeated state changes.

Solution: Developed a basic software debouncing routine using short delays (delay(50);) after detecting an initial state change, ensuring only deliberate user inputs are registered by the state machine.

C

Timing and Logic Flow

Challenge: The system must monitor continuous variables (gas levels, distance) simultaneously with asynchronous user inputs while managing the LCD update rate without blocking safety actuation loops.

Solution: Architected the loop as a structured state machine with distinct states (Auto, Manual, Emergency). The main sensor subroutines are kept non-blocking, ensuring safety loops check critical sensors every cycle regardless of user inputs.

System Workflow and Non-Blocking Logic

This section outlines the firmware's execution path and safety logic priorities.

System initializes LCD, sets pin modes, and pre-calibrates analog inputs
→
Loop begins: System aggressively polls safety critical sensors (Gas/Distance)
→
Firmware executes safety checks and immediately actuates alerts if thresholds are breached

Actuation Logic Subroutines

  • PWM Actuation: scaled lighting (inverse mapped LDR luminosity)
  • Immediate Danger Actuation: Buzzer (highest tone), Red LED (steady), LCD alert. Threshold breach in Gas (>450), Temp (>40), or Dist (<10).
  • Warning Actuation: Buzzer (alternating tone), Yellow LED (steady), LCD warning.
  • Status Update Subroutine: The 16x2 LCD character buffer is updated using standard polling at non-blocking intervals.

System State Machine

  • Auto State: Full autonomous monitoring and closed-loop actuation based on inputs.
  • Manual State: Actuators set to fixed safe behaviors; basic safety actuarion overrides manual control.
  • Emergency State: Forced alert state; buzzer and red LED prioritized.

This is the compiled source code that implements the complete system workflow.

/**************************************************************
 * Smart Autonomous Monitoring and Control System Firmware
 * Developed by Khablil
 * * Includes core functionality for State Machines, Non-Blocking Sensor 
 * Polling, PWM Closed-Loop Actuation, and LCD Interface management.
 * * Hardware Requirements: Arduino Uno, LCD 16x2, LDR, DHT11, 
 * MQ-2, Ultrasonic HC-SR04, Buzzer, Push Buttons, LEDs.
 **************************************************************/

#include <LiquidCrystal.h>
#include <DHT.h>

// Initialize Parallel LCD interface
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);

// DHT11 Sensor Configuration
#define DHTPIN A0
#define DHTTYPE DHT11
DHT dht(DHTPIN, DHTTYPE);

// Analog Input Definitions
const int ldrPin = A1;
const int gasPin = A2;

// Ultrasonic Sensor Definitions
const int trigPin = 6;
const int echoPin = 7;

// Actuator Pin Definitions
const int redLED = 8;
const int yellowLED = 9;
const int greenLED = 10;
const int pwmLED = 13;
const int buzzerPin = A3;

// User Input Pin Definitions
const int autoButton = A4;
const int manualButton = A5;
const int emergencyButton = A6;

// Global System Variables
int mode = 0;  // 0=Auto, 1=Manual, 2=Emergency
float temperature = 0;
int lightValue = 0;
int gasValue = 0;
long duration = 0;
float distance = 0;

void setup() {
  // Initialize LCD and splash screen
  lcd.begin(16, 2);
  lcd.print("Project by");
  lcd.setCursor(0, 1);
  lcd.print("Khablil");
  delay(2000);  // Simple splash screen delay
  lcd.clear();

  // Initialize sensors
  dht.begin();
  pinMode(ldrPin, INPUT);
  pinMode(gasPin, INPUT);
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  
  // Initialize actuators
  pinMode(redLED, OUTPUT);
  pinMode(yellowLED, OUTPUT);
  pinMode(greenLED, OUTPUT);
  pinMode(pwmLED, OUTPUT);
  pinMode(buzzerPin, OUTPUT);
  
  // Initialize buttons with internal pull-up
  pinMode(autoButton, INPUT_PULLUP);
  pinMode(manualButton, INPUT_PULLUP);
  pinMode(emergencyButton, INPUT_PULLUP);
  
  // Initialize serial for debugging if required
  Serial.begin(9600);
}

void loop() {
  // 1. Monitor Inputs (User and Environment)
  readButtons();
  readSensors();

  // 2. Execute State Machine Logic
  if (mode == 2) {
    emergencyMode();
  } else if (mode == 1) {
    manualMode();
  } else {
    autoMode();
  }

  // 3. Update Display Telemetry (Non-Blocking)
  updateDisplay();
  
  // 4. Set Loop Timing (Basic Debounce / Stabilizer)
  delay(300);
}

void readButtons() {
  // Polling inputs with software debouncing implemented via the loop delay
  if (digitalRead(autoButton) == LOW) { delay(50); if(digitalRead(autoButton)==LOW) mode = 0; }
  if (digitalRead(manualButton) == LOW) { delay(50); if(digitalRead(manualButton)==LOW) mode = 1; }
  if (digitalRead(emergencyButton) == LOW) { delay(50); if(digitalRead(emergencyButton)==LOW) mode = 2; }
}

void readSensors() {
  // Poll DHT11 (Digital temperature)
  temperature = dht.readTemperature();
  if (isnan(temperature)) temperature = 0; // Handle read errors gracefully

  // Poll analog inputs
  lightValue = analogRead(ldrPin);
  gasValue = analogRead(gasPin);

  // Poll Ultrasonic Sensor (Distance measurement subroutine)
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);

  duration = pulseIn(echoPin, HIGH);
  distance = duration * 0.034 / 2;
}

void autoMode() {
  // Task 2: Closed-Loop PWM Control
  // Map light level (0-1023) to analog output brightness (255-0)
  int brightness = map(lightValue, 1023, 0, 0, 255);
  brightness = constrain(brightness, 0, 255);
  analogWrite(pwmLED, brightness);

  bool danger = false;
  bool warning = false;

  // Task 4 & 6: Prioritized Safety Actuation based on threshold breaches
  // Establish thresholds for emergency, warning, or safe states
  if (gasValue > 450 || temperature > 40 || distance < 10) {
    danger = true;
  } else if (gasValue > 300 || temperature > 30 || distance < 20) {
    warning = true;
  }

  // Actuate prioritized outputs
  if (danger) {
    digitalWrite(redLED, HIGH);
    digitalWrite(yellowLED, LOW);
    digitalWrite(greenLED, LOW);
    tone(buzzerPin, 1000); // Continuous Alarm
  } else if (warning) {
    digitalWrite(redLED, LOW);
    digitalWrite(yellowLED, HIGH);
    digitalWrite(greenLED, LOW);
    // Non-blocking warning pulse
    tone(buzzerPin, 700);
    delay(150);
    noTone(buzzerPin);
  } else {
    digitalWrite(redLED, LOW);
    digitalWrite(yellowLED, LOW);
    digitalWrite(greenLED, HIGH);
    noTone(buzzerPin); // System safe state
  }
}

void manualMode() {
  // Acts as a standby or non-actuating manual interface state
  analogWrite(pwmLED, 150); // Set fixed brightness
  digitalWrite(redLED, LOW);
  digitalWrite(yellowLED, HIGH); // Steady warning indication
  digitalWrite(greenLED, LOW);

  // Basic manual-mode safety override loop
  if (gasValue > 500 || temperature > 45 || distance < 8) {
    tone(buzzerPin, 1200);
  } else {
    noTone(buzzerPin);
  }
}

void emergencyMode() {
  // Immediate forced alarm state
  analogWrite(pwmLED, 255);
  digitalWrite(redLED, HIGH);
  digitalWrite(yellowLED, LOW);
  digitalWrite(greenLED, LOW);
  tone(buzzerPin, 1500); // Prioritized high-pitch alarm
}

void updateDisplay() {
  // Refreshes character buffer; polling ensures non-blocking updates
  lcd.clear();
  
  // Render Sensor Data (Line 1)
  lcd.setCursor(0, 0);
  lcd.print("T:");
  lcd.print(temperature, 1); // Render one decimal place
  lcd.print("C ");
  lcd.print("L:");
  lcd.print(lightValue);

  // Render System State (Line 2)
  lcd.setCursor(0, 1);
  if (mode == 0) lcd.print("AUTO ");
  else if (mode == 1) lcd.print("MANUAL ");
  else lcd.print("EMERG ");

  lcd.print("D:");
  lcd.print(distance, 0); // Render distance (cm)
}
